# ai-output-redacter

[![CI](https://github.com/ollieb89/ai-output-redacter/actions/workflows/ci.yml/badge.svg)](https://github.com/ollieb89/ai-output-redacter/actions/workflows/ci.yml)
[![npm](https://img.shields.io/npm/v/ai-output-redacter)](https://www.npmjs.com/package/ai-output-redacter)
[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](LICENSE)

> **Stop AI leaks before they leave CI.**  
> Scan and redact sensitive content in AI-generated outputs before they get posted, committed, or published.

## The Threat Model

AI models can:
- **Hallucinate credentials** — invent API keys, JWTs, or tokens that happen to look real
- **Echo environment variables** — reflect `$OPENAI_API_KEY` from system prompts into output
- **Repeat training data** — surface real emails, phone numbers, or SSNs from pretraining
- **Leak internal infrastructure** — expose private IPs, internal hostnames, or service URLs

PR summaries, job logs, eval reports, and commit messages generated by AI are a **new leak surface** that no existing CI tool covers. `ai-output-redacter` plugs that gap.

## Quick Start

```yaml
- uses: ollieb89/ai-output-redacter@v1
  with:
    scan-paths: 'ai-outputs/'
    mode: report
```

## Three Modes

| Mode | Behavior |
|------|----------|
| `report` | Scan and report findings — never blocks or modifies (safe for all pipelines) |
| `redact` | Overwrite matched content with `[REDACTED:rule-name]` in scanned files |
| `enforce` | Fail CI if any match found (configurable with `fail-on`) |

## Built-in Detection Patterns

| Rule | Severity | Detects |
|------|----------|---------|
| `api-key-openai` | Critical | OpenAI API keys (`sk-...`) |
| `api-key-anthropic` | Critical | Anthropic API keys (`sk-ant-...`) |
| `api-key-github` | Critical | GitHub PATs (`ghp_`, `ghs_`, `gho_`, `ghu_`, `gpr_`) |
| `api-key-aws-access` | Critical | AWS access key IDs (`AKIA...`) |
| `api-key-aws-secret` | Critical | AWS secret access keys |
| `api-key-gcp` | Critical | GCP/Google API keys (`AIza...`) |
| `api-key-azure` | High | Azure SAS tokens |
| `bearer-token` | High | HTTP Authorization Bearer tokens |
| `private-key-pem` | Critical | RSA/EC/OPENSSH private key blocks |
| `jwt-token` | High | JSON Web Tokens |
| `pii-email` | Medium | Email addresses |
| `pii-phone` | Medium | Phone numbers (E.164, US format) |
| `pii-ssn` | Critical | US Social Security Numbers |
| `private-ip` | Low | Private IP ranges (RFC 1918) |
| `credit-card` | Critical | Credit card numbers (Luhn-validated) |

> **Safe output guarantee:** Rule outputs never echo raw matched content. All reports use masked previews: `sk-ant-***[18 chars]`

## Action Inputs

| Input | Description | Default |
|-------|-------------|---------|
| `scan-paths` | Comma-separated files/dirs to scan | `.` |
| `config-path` | Config file path | `.ai-output-redacter.yml` |
| `mode` | `report` \| `redact` \| `enforce` | `report` |
| `fail-on` | `any` \| `high` \| `none` (enforce mode only) | `any` |
| `scan-stdin` | Also scan piped stdin | `false` |
| `ignore-paths` | Comma-separated paths to skip | `` |

## Action Outputs

| Output | Description |
|--------|-------------|
| `violations-found` | `true` if violations found |
| `violation-count` | Total violations |
| `rules-triggered` | Comma-separated rule names |
| `report-path` | Path to JSON artifact |

## Config File

`.ai-output-redacter.yml`:

```yaml
mode: enforce
fail-on: any
scan-paths:
  - ai-outputs/
  - .github/summaries/
ignore-paths:
  - "*.test.ts"
  - node_modules/
disabled-rules:
  - private-ip     # disable low-severity rules
custom-rules:
  - name: internal-domain
    pattern: '\.internal\.mycompany\.com'
    severity: high
  - name: customer-id
    pattern: 'CUS-[0-9]{8}'
    severity: medium
```

## Full CI Example

```yaml
name: Scan AI Outputs

on: [push, pull_request]

jobs:
  scan:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Generate AI output
        run: node scripts/generate-summary.js > ai-outputs/summary.txt
      - uses: ollieb89/ai-output-redacter@v1
        with:
          scan-paths: 'ai-outputs/'
          mode: enforce
          fail-on: high
```

## CLI

```bash
npx ai-output-redacter src/
npx ai-output-redacter --mode enforce --fail-on any .
npx ai-output-redacter --format markdown ai-outputs/
```
