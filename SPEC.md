# ai-output-redacter — Spec

## What
GitHub Action that scans AI-generated outputs, logs, PR comments, summaries, and workflow artifacts for secrets, PII, tokens, and sensitive patterns — then redacts or blocks before they escape CI.

## Positioning
> Stop AI leaks before they leave CI.
> Scan and redact sensitive content in AI-generated outputs before they get posted, committed, or published.

## Problem
AI models hallucinate credentials, repeat training data, or inadvertently echo environment variables into outputs. PR summaries, job logs, eval reports, and commit messages generated by AI are a new leak surface that no existing CI tool covers.

## Features (MVP v1.0.0)

### Track 1 — Detection Engine
1. **Built-in Patterns** — detect out of the box:
   - API keys / tokens (OpenAI, Anthropic, GitHub, AWS, GCP, Azure patterns)
   - Bearer tokens / Authorization headers
   - Private keys (RSA, EC, PEM blocks)
   - Emails, phone numbers (E.164), SSNs
   - Internal hostnames / private IP ranges (10.x, 172.16.x, 192.168.x)
   - Credit card numbers (Luhn check)
   - JWT tokens
2. **Custom Rules** — user-defined regex patterns via config
3. **Redaction Engine** — replace matches with `[REDACTED:rule-name]`
4. **Multi-source Scanner** — scan: files, directories, stdin, step outputs, log snippets

### Track 2 — Product Layer
5. **Config Schema** — `.ai-output-redacter.yml`: rules, ignore-paths, mode, fail-on
6. **Three Modes**:
   - `report` — scan and report findings, never block
   - `redact` — overwrite matched content in files, continue
   - `enforce` — fail CI if any match found
7. **Safe Output** — NEVER echo raw matched content. Show only: rule name, location, masked snippet (`sk-ant-...***`)
8. **PR Comment** — summary table: rule triggered, file/source, count, masked preview
9. **Job Summary** — scan totals, rules triggered, redaction count

## Important design constraint
**Never echo full matched content.** Output format:
```
⚠️  api-key-anthropic  →  ai-review-output.txt:L12  →  sk-ant-***[18 chars]
```
The tool must never become the leak vector.

## Action Inputs
- `scan-paths` — files/dirs to scan (default: `.`)
- `scan-stdin` — also scan piped input (default: false)
- `config-path` — config file (default: .ai-output-redacter.yml)
- `mode` — report | redact | enforce (default: report)
- `fail-on` — any | high | none (default: any, in enforce mode)
- `ignore-paths` — glob patterns to skip

## Action Outputs
- `violations-found` — true/false
- `violation-count` — total matches
- `rules-triggered` — comma-separated rule names
- `report-path` — path to JSON report artifact

## Config Example
```yaml
# .ai-output-redacter.yml
mode: enforce
fail-on: any
scan-paths:
  - ai-outputs/
  - .github/summaries/
ignore-paths:
  - "*.test.ts"
  - node_modules/
custom-rules:
  - name: internal-domain
    pattern: '\.internal\.mycompany\.com'
    severity: high
  - name: customer-id
    pattern: 'CUS-[0-9]{8}'
    severity: medium
```

## Architecture
- TypeScript GitHub Action + CLI (npx ai-output-redacter)
- src/: index.ts, scanner.ts, matcher.ts, redactor.ts, reporter.ts, config.ts
- src/patterns/: builtin.ts (all built-in detection rules)
- bin/cli.ts
- tests/ (Jest, 35+ tests — cover each pattern, redactor, config, reporter, safe output)

## Critical Tests
- Each built-in pattern must have positive AND negative test cases
- Redaction must never output raw matched content in any test
- Masked preview format: first 6 chars + *** + [N chars]
- Custom rules must compose with built-ins without conflict
